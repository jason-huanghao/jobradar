# Using JobRadar with OpenClaw and Claude

*Your job search agent — zero-friction setup guide*

---

## What you need before starting

| Requirement | Minimum |
|------------|---------|
| Python | 3.11+ |
| LLM API key | Any one: OpenAI / DeepSeek / Volcengine Ark / Z.AI / Ollama (local) |
| Your CV | A file path **or** a URL (PDF, Markdown, HTML, DOCX) |

That's it. No database, no Docker, no separate backend.

---

## Option A — Using JobRadar inside OpenClaw

OpenClaw reads the `SKILL.md` file automatically and makes all commands available as conversational actions.

### 1. Install

```bash
git clone https://github.com/jason-huanghao/jobradar.git
cd jobradar
pip install -e .
```

Place the `jobradar/` folder in your OpenClaw skills directory (or any directory OpenClaw scans).

### 2. One-command bootstrap

You can set up entirely from a conversation, without editing any files:

```
You:  Set up JobRadar. My CV is at https://mysite.com/cv.pdf, 
      I'm looking for AI/ML jobs in Berlin and Hamburg, 
      and my OpenAI key is sk-xxxx
```

OpenClaw will run:
```bash
jobradar --init \
  --cv https://mysite.com/cv.pdf \
  --locations "Berlin,Hamburg,Remote" \
  --llm openai \
  --key sk-xxxx
```

This writes `config.yaml` and `.env` automatically — no wizard, no manual YAML editing.

### 3. Verify setup

```
You:  Check if JobRadar is ready
```

OpenClaw runs `jobradar --health --json` and reports:
- ✅ LLM connected (with latency)
- ✅ Job sources reachable
- ✅ CV found
- ⚠️ Any issues (missing cookie, unreachable source, etc.)

### 4. Run your first job search

```
You:  Find me AI jobs now
```

→ OpenClaw runs `jobradar --mode quick` (fast: 2 sources, ~3 min)

```
You:  Run the full job search
```

→ OpenClaw runs `jobradar --update` (all sources, incremental)

### 5. See results

```
You:  Show me today's top jobs
```

→ OpenClaw calls `jobradar --show-digest --json` and presents the results inline.

```
You:  What's in my job pool?
```

→ OpenClaw calls `jobradar --status --json`:
```json
{
  "pool": { "total": 312, "scored": 298, "unscored": 14 },
  "top_job": { "title": "ML Engineer", "company": "DeepL", "score": 8.4 },
  "sources": { "arbeitsagentur": "enabled", "indeed": "enabled" }
}
```

### 6. Act on results

```
You:  Write a cover letter for the DeepL job
```
→ `jobradar --generate-app "DeepL"`

```
You:  I applied to Zalando yesterday
```
→ `jobradar --mark-applied "Zalando"`

```
You:  Why did Databricks score low?
```
→ `jobradar --explain "Databricks"` — shows full dimension breakdown + LLM reasoning

```
You:  I liked that AMD role, remember that for future searches
```
→ `jobradar --feedback "AMD liked"`

### 7. Automate daily runs

```
You:  Set up JobRadar to run every morning
```
→ `jobradar --install-agent` — installs a launchd (macOS) or cron (Linux) job at 8am

## Option B — Using JobRadar with Claude (claude.ai or Claude Code)

### Claude Code (recommended)

Claude Code reads `CLAUDE.md` from the project root. All commands work directly in the terminal.

```bash
# In Claude Code terminal:
cd jobradar
jobradar --health           # verify setup
jobradar --status           # see current pool state
jobradar --mode quick       # fast 3-minute test
jobradar --update           # daily incremental run
```

Ask Claude Code naturally:
```
"Run a quick job search and show me the results"
"My CV is at ~/Downloads/resume.pdf — set up JobRadar and find ML jobs in Munich"
"Generate a cover letter for the Siemens job"
```

Claude Code will translate these to the right `jobradar` commands automatically.

### Claude (claude.ai with Desktop Commander or filesystem MCP)

If you have the Desktop Commander or Filesystem MCP connected to Claude:

**First-time setup:**
```
"Set up JobRadar for me. My CV URL is https://… and I have an OpenAI key"
```
Claude will run:
```bash
jobradar --init --cv https://… --llm openai --key sk-… --locations "Berlin,Remote"
jobradar --health
```

**Daily usage:**
```
"Run my job search"          → jobradar --update
"What are my top matches?"   → jobradar --show-digest --json
"Apply to Bosch mentally"    → jobradar --mark-applied "Bosch"
```

## Minimum required configuration

The `--init` command creates the smallest valid `config.yaml`. For most users this is all that's needed:

```yaml
# config.yaml — auto-generated by jobradar --init
candidate:
  cv_url: "https://mysite.com/cv.pdf"   # OR cv_path: "./cv/cv.md"

llm:
  text:
    provider: "openai"
    model: "gpt-4o-mini"
    api_key_env: "OPENAI_API_KEY"     # key in .env file

search:
  locations: ["Berlin", "Hamburg", "Remote"]

sources:
  arbeitsagentur: { enabled: true }
  jobspy: { enabled: true, boards: ["indeed", "google"], country: "germany" }
```

**That's 13 lines.** Everything else has sensible defaults.

The `.env` file (same directory):
```
OPENAI_API_KEY=sk-your-key-here
```

## Common one-liners

```bash
# First-time setup (non-interactive, agent-friendly)
jobradar --init --cv ./cv.md --llm deepseek --key $DEEPSEEK_API_KEY --locations "Berlin,Remote"

# Verify everything works
jobradar --health

# Fast test run (2 sources, ~3 min)
jobradar --mode quick

# Daily incremental update (new jobs only)
jobradar --update

# See current state as JSON (for agents to parse)
jobradar --status --json

# See top jobs as JSON (for agents to present)
jobradar --show-digest --json

# Generate cover letter
jobradar --generate-app "Company Name"

# Mark applied + record feedback
jobradar --mark-applied "Company Name"
jobradar --feedback "Company Name liked"

# Install daily 8am automation
jobradar --install-agent
```

## Filtering noise without touching the LLM

Add to `config.yaml` to drop internships and former employers before scoring:

```yaml
search:
  exclude_keywords: ["Praktikum", "Werkstudent", "internship", "Ausbildung"]
  exclude_companies: ["MyFormerEmployer"]
```

These filters run before the LLM — they're free and fast.

## Adding Chinese job platforms

If you're also targeting China-based roles:

```yaml
search:
  locations: ["Berlin", "Remote", "上海", "北京"]

sources:
  zhilian: { enabled: true }              # No auth — works immediately
  bosszhipin: { enabled: true }           # Needs one-time browser cookie setup
  lagou: { enabled: true }                # Auto-fetches session cookies
```

> **Note:** Chinese platforms return best results from a Chinese IP. From Europe, use a VPN or run JobRadar on a CN cloud server.

**BOSS直聘 cookie setup (one-time, ~2 min):**
1. Log in at [zhipin.com](https://www.zhipin.com) in Chrome
2. DevTools → Application → Cookies → `www.zhipin.com`
3. Copy `__zp_stoken__` and `wt2` values
4. `export BOSSZHIPIN_COOKIES="__zp_stoken__=xxx; wt2=yyy"` in `.env`

## LLM provider quick-reference

| Provider | Env var | Notes |
|----------|---------|-------|
| OpenAI | `OPENAI_API_KEY` | `gpt-4o-mini` is cost-effective |
| DeepSeek | `DEEPSEEK_API_KEY` | Cheapest option (~$0.14/1M tokens) |
| Volcengine Ark | `ARK_API_KEY` | Best for CN users (doubao models) |
| Z.AI | `ZAI_API_KEY` | Z.AI coding plan |
| OpenRouter | `OPENROUTER_API_KEY` | 200+ models, one key |
| Ollama | *(none)* | Fully local, auto-detected |

All providers auto-detected from environment — just set the key, no config change needed.

## Troubleshooting

| Symptom | Fix |
|---------|-----|
| `config.yaml not found` | Run `jobradar --init` or `jobradar --setup` |
| LLM error | Run `jobradar --health` to diagnose |
| 0 jobs from BOSS直聘 | Set `BOSSZHIPIN_COOKIES` env var |
| CN platforms return 0 | Expected from EU IPs — use VPN or CN server |
| Scoring very slow | Reduce `scoring.batch_size` in config; or switch to a faster model |
| Too many internship results | Add `exclude_keywords` to config |

## Full command reference

```
jobradar --init            Non-interactive first-time setup
jobradar --setup           Interactive setup wizard
jobradar --health [--json] Connectivity + config check
jobradar --status [--json] Pool stats + source readiness
jobradar                   Full pipeline
jobradar --update          Incremental (new jobs only) + email
jobradar --mode quick      Fast test (BA + Indeed, ~3 min)
jobradar --mode dry-run    Show queries, don't execute
jobradar --show-digest [--json]   Today's top jobs
jobradar --generate-app COMPANY   Cover letter
jobradar --mark-applied COMPANY   Mark as applied
jobradar --explain COMPANY        Score breakdown
jobradar --feedback "X liked"     Record preference
jobradar --install-agent   Daily 8am automation
jobradar --cv PATH_OR_URL  Override CV for this run
jobradar --rerun-scoring   Re-score all jobs from scratch
```
